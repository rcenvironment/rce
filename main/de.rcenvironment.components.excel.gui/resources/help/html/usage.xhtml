Excel Component Usage

SynopsisThe Excel Component to get access to Microsoft Excel files in RCE with input and output cells and macro execution.

Rationale

This component wraps an existing Microsoft Excel file which is linked to RCE. The general principle is:

1.      Copy of the existing Excel file as temporarily file in a temporarily folder (working copy)

2.      Waiting for all input channels which are needed to run the component (depending on its “usage” type)

3.      Executing VBA-macro “Before Excel run”

4.      Copy all input channels to their specific cell ranges. It there are multiple values in an input channel the first value occurring (“first in, first out”) will be copied

5.      Updating all formulas

6.      Executing VBA-macro “After input variables are written”

7.      Updating all formulas

8.      Reading all output channel values from their specific cell range9.      Executing VBA-macro “After Excel run”10.  Deletion of temporarily Excel file

Usage

The Excel Component has two (resp. three) views to be configured.

File

First, there is the file view, where you can link a specific Excel file to the RCE component.

Inputs/Outputs

The second view is for creating the inputs and outputs of the Excel component. The configuration of the channels of both types is (mostly) similar. A RCE-channel is always connected to a specific Excel cell range. The button “Autodiscover” discovers automatically all input and output channels which are described as user-specific cell areas which a beginning “I_” for input-channels and a beginning “O_” for output channels.

 

Following a short description of all channel-configuration parameters:

Element

Description

Name

The name of the RCE-channel

Data Type

There are five different data types which represent Excel and RCE types:

Short Text: alphanumeric text
Boolean: true/false
Integer
Float: floating-point number
Small Table: cell (area)

Constraints

See RCE user guide

Handling

See RCE user guide

Address

Address represents Excel cell areas, e. g. “Table1!A1:B5”. Alternatively, in Excel where might be a user-specific name of a cell area which can be used at this point

Expand Cell Area (only input channels)

If the user does not know the size of the cell area which she wants to insert, this field can be set to true. Now the upper left cell area field can be selected in “Address”-parameter. From that address on the complete table will be inserted, ignoring existing cell entries

Delete empty entries (only output channels)

If a selected cell area contains empty rows at the end these will be cut off when setting this field to true.

Button "Select address in Excel file...

To select a cell area in Excel this button opens the file in Microsoft Excel so the user can select a specific cell area.

When you are finished adding variables, you can connect them in the connection editor.

Macros

A third view is for configuring VBA-macros for executing during runtime.

While in Microsoft Excel properties the access to the VBA-project object model is granted all available VBA-macros can be discovered with button “Discover Macros”. All available macros can be chosen in each dropdown menus.

Requirements

·        Microsoft Excel should be installed. RCE component is tested with Microsoft Excel 2003 and 2010

·        For an automatic detection of VBA-macros the VBA-project must be trusted. Please start Excel as administrator. Under file->options->trust center->trust center settings->macro settings->trust access to the VBA project object model->activate
